## §PL-10 State & write idioms

Reads: raw `fetch(getBackendUrl() + '/api/advertising/…', { cache: 'no-store' })` — no SWR/react-query anywhere. Writes: PATCH JSON `{ ...fields, applyImmediately: true|false, reason: '<surface> <action>' }` (Amazon-side convention; the eBay side keeps its guarded-write body shapes). **No optimistic updates** — write completes → parent `onSaved?/onRefresh?` → full reload. Bulk = `Promise.all` per-id PATCHes, failures counted into toast text ("… N failed (write-gate / non-live)"). Toasts: grid pages = fixed `.h10-am-toast` auto-cleared 5-6s; detail form = inline `span.msg` in the sticky footer 3.2s. Every async button holds a busy boolean, disables, and swaps label ("Saving…/Applying…/Launching…").
