# EPO — Orders at enterprise grade (proposal · gate 1)

> Sources: F0-TEARDOWN verdicts (citable law), a fresh 2026-07-11 external sweep (SAP S/4HANA fulfillment cockpit, D365, NetSuite, Odoo 17/18, ERPNext, Katana, MRPeasy, Fulcrum, Cetec, JobBOSS², Infor SyteLine, Plex, ApparelMagic, Uphance, Printavo + Stripe/ShipStation/HubSpot/Linear patterns), a second sweep on order-to-cash linkage / brand-scoped views / zero-communication-gap patterns (Odoo chatter, NetSuite customer center, Shopify Timeline, Front, Printavo), and a line-level audit of the shipped FP4 implementation — the full element-by-element evidence is `EPO-UI-INVENTORY.md`. Nothing here is built; each phase re-gates.

## §1 What /orders IS

The operational board — every confirmed job's single truth from "quote accepted" to "closed and reviewed." Its loyalties, in order:
1. **The state machine is the truth.** One authority for every transition, no path around it, no silent change — ERPNext docstatus verdict (ADOPT) and the program invariant.
2. **The order is the hub of the created-from chain.** Email → quote → order → work orders → shipment → invoice → payment → review, every hop a link both ways (NetSuite chain verdict: ADAPT, extended upstream to the email).
3. **Money and promises are visible where the work is.** The Owner should never leave the board to answer "is this paid?" or "will this be late?"

## §2 What exists (verified)

FP4 board (grid + kanban with validated @dnd-kit drags, 3 counters, status tabs, party filter, search), the ONE-TIMELINE (10 event kinds), Start-production with WO explosion + FD13 deposit gate, payments recording with WO unblock, promise-date edit, cancel+reason with WO cascade, size-run matrix, streamed CSV export, `canTransition` state authority, FS1 query bounds (kanban lanes cursored, true counts). FP6/FP8/FP9 later delivered the real lifecycle drivers (auto-READY, label-buy→SHIPPED, tracking→DELIVERED, invoices/rollup) — **but the orders page was never re-wired to consume them.** That drift is the story of this audit.

## §3 What the audit found (defect register — evidence in `EPO-UI-INVENTORY.md`)

**Correctness:**
- **C1** — Stale FP8 stopgap still live: grid menu + kanban drop mark orders SHIPPED via a manual modal whose copy still says "labels arrive in FP8", bypassing the real shipment flow; kanban SHIPPED lane is a legal drop target. Violates F0-IA "cannot drop into SHIPPED without a shipment."
- **C2** — The FP6/FP8 drivers (stage-done→READY, buy→SHIPPED, tracking→DELIVERED, void→READY) write `order.state` directly, bypassing `canTransition`; the void edge SHIPPED→READY isn't even in the graph. Single-authority holds only for `/api/orders` routes.
- **C3** — Silent mutations: promise-date change and line edit publish **no event**; WO cascades (cancel/unblock/create) write **no per-WO audit rows**. Direct violation of "no silent state changes."
- **C4** — Payments POST has no idempotency guard: a double-submit records the payment twice (money!).
- **C5** — Global search/⌘K emits `?focus=` for every entity; only inbox honors it. Every palette jump to an order/quote/contact/product lands on the page but never opens the record.
- **C6** — Grid silently caps at 200 (API cursor exists, UI ignores it) — orders past 200 vanish from the list view.
- **C7** — Net column renders €0,00 for grain-stripped viewers instead of "—" (keys off `lineCount`, not field presence).
- **C8** — An order with no `bornFromQuote` gets `depositRequired=0`: the FD13 gate switches off silently instead of surfacing "no deposit terms."
- **C9** — No optimistic concurrency anywhere: read-then-write PATCH, last-write-wins (two sessions can cross-edit promise/state).

**Enterprise gaps:**
- **E1** money truth invisible (balance/invoiced/paid/actual margin live in the FP9 rollup, never surfaced here) · **E2** dead-end navigation (7 of 10 timeline kinds unlinked; WO/party/invoice plain text; ConvertBar backlink static with FP4-era copy; inbox order-pill discards the id; bell has no order emitter) · **E3** no exception queue (no "needs attention" view) · **E4** no promise integrity (no original promise, slip history, or pre-late at-risk flag; timeline omits promise changes and the cancel reason) · **E5** no amendments after CONFIRMED (only size-run, only while CONFIRMED; no add/remove line, no revisioning) · **E6** no partial shipment for size-run B2B · **E7** no returns/rework record · **E8** no customer notification on any state change — the communication gap, on the page that IS the lifecycle · **E9** no bulk ops, saved views, or date filter; export ignores filters · **E10** comments + attachments exist in schema, unwired on orders · **E11** zero SSE consumption (analytics listens to `order.updated`; the orders board doesn't) · **E12** DS/a11y debt (no skeletons, no keyboard path on kanban, hand-rolled tabs/toggle/banners/inputs, 3 date formats, duplicated cancel modals, Back-button trap) · **E13** no historical order import · **E14** no party-scoped URL — a "brand view" has nothing to deep-link to.

## §5 The phases

- **EPO.1 — State-machine truth & write integrity** (kills C1 C2 C3 C4 C8 C9). One `transition-service` = the only writer of `order.state` — `canTransition` + audit + durable event + timeline row in one transaction; FP6/FP8 drivers route through it; void edge joins the graph; kanban SHIPPED lane becomes non-droppable (drop → "Buy label" CTA); stale modal deleted; per-WO audit rows on cascades; `promise-changed`/`line-edited` become events; payments gain an idempotency key; PATCH carries an `updatedAt` guard (409 on stale); orders without deposit terms show "no deposit terms from quote" instead of a silently-off gate. **Exit:** every order mutation provably audited + published (test-enforced); illegal/stale/duplicate writes 409/422; harness green.
- **EPO.2 — Money on the order** (kills E1 C7). Detail money strip consuming `orderFinancials` (Quoted · Invoiced · Paid · **Balance** + margin `est→actual`), deposit tracker (requested→invoiced→paid), invoices listed on the order with PDF/send/mark-paid via existing FP9 endpoints, one payment badge + balance on list rows, low-margin flag, overdue-party banner (non-blocking), stripped money renders "—" never €0,00. **Exit:** numbers byte-identical to the financials drawer (same fold, parity-tested); Worker sees none of it (strip test).
- **EPO.3 — The connected order** (kills C5 E2 E11). Chain chips (Quote · WOs · Shipments · Invoices · Payments · Returns) with counts; every timeline kind gets its href; timeline gains invoices, promise changes (+reason), cancel reason, stage completions, tracking/delivered; party names link `/contacts?c=`; WO rows link `/production?wo=` (reader per D-1); search route emits the honored params; bell notifications for order events emit `/orders?o=`; board + detail go live via `useFactoryEvents` (FS2 shipped). **Exit:** zero plain-text entity dead-ends on /orders; a ⌘K jump opens the order; a transition in one window updates the board in another without refresh.
- **EPO.4 — Promise integrity + the exception cockpit** (kills E3 E4). Additive `Order.originalPromiseDateAt` (backfilled from quote/current); slip history derived from audit + "slipped ×N" badge; at-risk fold (days remaining vs remaining stages × FP10 stage lead-times); **Needs attention** default tab — late · at-risk · deposit unpaid · stalled WO — each row with its one-click next action; date-range filter. Aggregates (on-time-vs-original %) exposed as a fold for EPA, not built here. **Exit:** cockpit shows only actionable rows, each with an action; original promise immutable; at-risk fires before lateness on the harness.
- **EPO.5 — Lifecycle depth: amendments · partial shipment · returns** (kills E5 E6 E7). Amend-after-confirm: `OrderRevision` snapshot + field diff in the timeline + price delta vs confirmed; net-changing amendments void the acceptance and re-request approval through the thread (consumes EPQ's acceptance-evidence pattern). Partial shipment: per-size remaining in the buy flow + explicit ship-remainder/cancel-remainder; derived "Partially shipped x/y" badge (no new lifecycle state, per D-2). Returns: per-line repair/remake/credit; repair/remake spawns a rework WO through the normal stage flow; credit recorded against the order (EPF consumes it). **Exit:** nothing confirmed is silently editable (docstatus verdict, test-enforced); a size-run order round-trips 80-now/40-later; a return produces a QC-gated rework WO.
- **EPO.6 — Zero communication gap** (kills E8 E10). Transition-triggered customer drafts into the order's own Gmail thread (FP1 MIME pipeline): confirmed · deposit received · shipped+tracking · payment settled, plus per-party opt-in "in production"; send-on-confirm default, **skip is logged to the timeline**; dual composer on the detail (Internal note via the existing Comment service — amber, distinct — vs Email customer); attachments (FileDropzone → Attachment rows); OrderDetail gains the tab host FC1 expects (Space seam). **Exit:** every state change lands in the thread or its skip is recorded; internal/external/system entries interleave in one timeline; no automated send in tests (sandbox dry-run).
- **EPO.7 — The workbench** (kills C6 E9 E12 E13 E14). FS3 adoption (assigned): VirtualDataGrid for the grid (kills the 200-cap) + AsyncCombobox party filter; saved views as pinned tabs with one default; `selectable` + BulkActionBar (bulk status/export); `?party=` deep link + dismissible "filtered to {Brand} ×" chip (the brand-view contract, D-5); export honors active filters; historical order CSV import (dry-run idiom); DS sweep — Tabs/SegmentedControl/MetricStrip/Banner/EmptyState/Input/Skeleton/Stepper, tokens over hex, one date format, one shared cancel/transition module; keyboard: j/k/Enter/e core loop + KeyboardSensor or menu-parity for kanban; pushState so Back closes the detail. **Exit:** 50k-order harness green through the virtual grid; core loop drivable by keyboard; ds-parity + zero raw hex on the page.
