# studies/

The research hub. Two kinds of document live here:

1. **Feature dossiers** — `NN-<feature>.md`. One per surface (ads/campaigns,
   products, orders, fulfillment, pricing…). What it does, how data flows, which
   components it uses, cross-platform parity, gaps, and an upgrade/automation
   roadmap. These feed both the migration (Phase 9) and future feature work.
2. **Cross-platform analyses** — how a capability behaves across Amazon, eBay,
   and Shopify (the active channels), and where Nexus's model aligns or diverges.

This is where "conduct studies on each feature" and "study how it all works
across platforms" live. The component work and the studies are decoupled — the
studies can grow independently once the framework (Phase 8) is in place.

## How to add a study

Copy `_TEMPLATE.md` → `NN-<feature>.md`, fill it in, add a row below.
